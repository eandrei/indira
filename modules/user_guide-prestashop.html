1 Introduction
AwoCoupon Pro is a coupon system that allows you to do many different things not part of the basic coupon system. Here are a few of its features:
  • Product(s) specific discounts
  • Category(s) specific discounts
  • Manufacturer(s) specific discounts
  • Vendor(s) specific discounts
  • Customer(s) specific discounts
  • Shopper Group(s) specific discounts
  • Shipping Coupons
  • The ability to only trigger coupons if a minimum cart total is reached
  • The ability to set expiration dates
  • The ability to restrict the number of times a coupon can be used by either a single user or in total
  • Easy management of all coupons
  • Gift Certificates
  • History of Uses
  • Import feature
  • Generate coupons
  • Reports
  • .... and more

3 Using the software - Administration
Now that we have finished the installation, we can look closely at the features presented in this application.
3.1 Getting Around
Log into the administration screen and go to Payment » AwoCoupon. Here you will see the control panel. It shows the general statistics of what coupons you currently have configured as well as a list of menus. From here you can enter a new coupon, view configured coupons, manage imports, view reports, and a variety of other features.
3.3 Coupons
This screen shows a list of coupons that have been created. At the top, there is a list of filters and a search field to help you find coupons efficiently. A coupon can be published, unpublished, deleted, or edited from this screen.
3.2.1 New Coupon (fields)
To add a new coupon, select "New Coupon" from the control panel, or "New" from the coupons list.
Fields:
3.2.1.1 Function Type
Options
Coupon: used as a normal coupon.
Gift Certificate: used as a gift certificate for products only.
Shipping: create coupons for shipping
Buy X Get Y: special promotion coupons such as buy 1 get 1 free, buy 3 for 2, etc
Parent: create coupons for other coupons
3.2.1.2 Coupon Code
Enter a unique coupon code of up to 32 characters
3.2.1.3 Published
Options
Published: active coupon
Unpublished: inactive coupon
Template: inactive coupon used in the generation of coupons in external plugins
3.2.1.4 Percent or Amount
Not used with gift certificates
Options
Percent: coupon takes a percentage of value
Amount: coupon is an actual figure that is subtracted from total
3.2.1.5 Discount Type
This option can only be used when an asset is selected, otherwise it should be set to "Overall".
Options (Shipping)
Overall: The shipping discount is given if any of the products from the product list are in the cart. If no product is selected, then it is assumed all the products are valid
Specific: The shipping discount is given if all the products in the cart are also in the coupon's product list.
Options (Everything Else)
Overall: The discount is calculated as a percentage of the overall total in the shopping cart
Specific: The discount is calculated as a percentage of the total of the products configured in the coupon that are in the shopping cart
3.2.1.6 Value
Number: this is the value of the coupon and is dependent on the Percent or Amount field to determine how it is calculated
3.2.1.7 Value Definition
Not used with gift certificates or function type 2 of shipping.
Defined input: this field is used instead of the value field if you want to create progressive coupons. Progressive coupons are coupons that are based on the number of products customers buy. For example, you could have a progressive coupon that says if a customer buys 1 item, then give him/her a 10% discount. But if the customer buys 2 items then give him/her a 20% discount... etc etc. Using the example just mentioned, the input would look like 1-10;2-20;. You can also set the maximum number of products that get a percent off. Using the same example, lets say you want to give a 20% discount up to 10 items and then no more discounts. This can be done with the following value: 1-10;2-20;11-0;. The input field is very specific. As a definition this can be broken down to [quantity]-[value];[quantity]-[value];...etc etc.

By default the product count is done on all products selected. For example if you have products A, B, and C, then 2 A's are treated the same as 1 A and 1 B, in terms of the count. By clicking the checkbox 'Apply Distinct Count', then 2 A's are treated as 1 product while 1 A and 1 B are treated as 2 products.
3.2.1.8 Number of uses (Total)
Not used with gift certificates
Number: the number of uses as a total across all users. If left blank, there is no limit.
3.2.1.9 Number of uses (Per Customer)
Not used with gift certificates
Number: the number of uses as per customer. If left blank, there is no limit.
3.2.1.10 Minimum Value
Not used with gift certificates
Options
Overall: tests the minimum value on the order total
Specific: if selected specific assets (product, category, vendor, etc), then test the minumum value only on the those items
Number: this is the minimum value of the product total in the shopping cart before a coupon can take effect. If set to 0 or left blank, then there is no minimum value.
3.2.1.11 Start Date
Date/Time: the date/time the coupon/gift certificate begins to be active. If entered has to be in the format YYYY-MM-DD hh:mm:ss. If left blank, then it is valid from the time it is entered.
3.2.1.12 Expiration
Date/Time: the date/time the coupon/gift certificate expires. If entered has to be in the format YYYY-MM-DD hh:mm:ss. If left blank, then it never expires.
3.2.1.13 Don't Mix Products
Checkbox: used in buy x get y for the product selections. If checked, then Buy X product has to be the same as Get Y product. If not checked, then can mix and match provided products are on the buy x get y lists.
3.2.1.14 Automatically add to cart 'Get Y' product
Checkbox: used in buy x get y. If checked, AwoCoupon will attempt to add product Get Y to the shopping cart. Note it may not be able to for any number of reasons including the product is out of stock or the product requires attributes to be selected.
3.2.1.15 Exclude Products on Special
Checkbox: If checked, the coupon discount will not apply to products that are on special or discounted.
3.2.1.16 Exclude Gift Certificate Products
Checkbox: If checked, the gift certificate discount will not apply to products that are tagged as gift certificate products.
3.2.1.17 Customer Type
Not used with gift certificates
Options
Customer: create coupons for specific customer(s)
Shopper Group: create coupons for specific shopper group(s)
3.2.1.18 Customers
Not used with gift certificates
Multiple selection: shows all your customers. Selection of customers will activate the coupon only for those customers.
3.2.1.19 Shopper Group
Not used with gift certificates
Multiple selection: shows all your shopper groups. Selection of shopper groups will activate the coupon only for those customers within a shopper group.
3.2.1.20 Asset Type - Products
Options
Include: treats the selection as inclusive, allowing those items within the coupon
Exclude: treats the selection as exclusive, excluding those items from the coupon
Multiple selection: shows all your products.
3.2.1.21 Asset Type - Categories
Options
Include: treats the selection as inclusive, allowing those items within the coupon
Exclude: treats the selection as exclusive, excluding those items from the coupon
Multiple selection: shows all your categories.
3.2.1.22 Asset Type - Manufacturers
Options
Include: treats the selection as inclusive, allowing those items within the coupon
Exclude: treats the selection as exclusive, excluding those items from the coupon
Multiple selection: shows all your manufacturers.
3.2.1.23 Asset Type - Vendors
Options
Include: treats the selection as inclusive, allowing those items within the coupon
Exclude: treats the selection as exclusive, excluding those items from the coupon
Multiple selection: shows all your vendors.
3.2.1.24 Asset Type - Shipping
Only used with function type of Shipping
Options
Include: treats the selection as inclusive, allowing those items within the coupon
Exclude: treats the selection as exclusive, excluding those items from the coupon
Multiple selection: shows all your shipping.
3.2.1.25 Asset Type - Coupons
Only used with function type of Parent
This option allows you to join multiple coupons into one coupon by adding them as assets. When a customer uses a parent coupon, then depending on the process type (see below) the coupon is processed as such.
Options (Process type):
First found match
Lowest value coupon
Highest value coupon
All that apply
ONLY if all apply
Multiple selection: shows all coupons that are not function type 2 of parent.
3.2.1.26 Shop Restriction
This option is only available for Prestashop 1.5.x that have multistore enabled
This option gives you a list of shops in your multi store and can be used to restrict a coupon to one or more shops.
Multiple selection: shows all shops in your multistore.
3.2.1.27 Description
Text: Internal note about the coupon
3.2.2 Generate Coupons
To generate coupons, select the "Generate Coupons" button from the coupon list page.
Fields:
3.2.2.1 Template
Options
Single selection: Shows all the cuopon templates created within AwoCoupon. To create a new template, add a new coupon and under the 'Published' dropdown, select 'Template'.
3.2.2.2 Number
Number: this is the value for the number of coupons you want to create using the template coupon selected in 3.2.2.1.
3.3 New Gift Certificate Product (fields)
3.3.1 New Gift Certificate Product (fields)
Setting up a gift certificate product is the final step to creating a sellable gift certificate. Before doing this step, make sure you setup a product within the catalog which you wish to sell as a gift certificate. Once that is setup, you can then add a new gift certificate product within AwoCoupon. This means that when the gift certificate product is purchased and the payment has been confirmed, the code(s) of the gift certificate(s) purchased is/are automatically generated and emailed to the customer.

To add a new gift certificate product, select "Add New" from the Gift Certificate Products list.
Fields:
3.3.1.1 Product
Options
Single selection: a textbox, which allows you to start typing the product name. After a few letters, a dropdown is displayed of possible products which can be selected.
3.3.1.2 Template
Options
Single selection: shows a list of all coupon templates created within AwoCoupon.
3.3.1.3 Profile Image
Options
Single selection: shows all gift certificate profiles setup within AwoCoupon.
3.3.1.4 Published
Options
Published: active gift certificate product
Unpublished: inactive gift certificate product
3.3.1.5 Expiration
Options
Days: sets the expiration x days from when the gift certificate was created
Months: sets the expiration x months from when the gift certificate was created. Assumes 30 days in a month.
Years: sets the expiration x years from when the gift certificate was created. Assumes 365 days in a year.
Number: if both the dropdown and textfield are left blank, there is no expiration.
3.3.1.6 Vendor Name
Text: Optional. The name of the vendor who will redeem the voucher. Used to send an email to the vendor if enabled in configuration.
3.3.1.7 Vendor Email
Text: Optional. The email of the vendor who will redeem the voucher. Used to send an email to the vendor if enabled in configuration.
3.3.1.8 Recipient Name ID
Number: Optional. If the customer can enter a name for the voucher, then the custom field will have to be set up. This can be setup within the product that is being sold under the customization tab. The value entered here is the number shown in front of the text field for entering a name.
3.3.1.9 Recipient Email ID
Number: Optional. If the customer can enter a email for the voucher, then the custom field will have to be set up. This can be setup within the product that is being sold under the customization tab. The value entered here is the number shown in front of the text field for entering a email.
3.3.1.10 Recipient Message ID
Number: Optional. If the customer can enter a message for the voucher, then the custom field will have to be set up. This can be setup within the product that is being sold under the customization tab. The value entered here is the number shown in front of the text field for entering a message.
3.3.2 Codes
There are certain cases were you would like to choose your own codes. AwoCoupon Pro gives you this ability. Using a csv file in the format specified, you can upload your own codes that will be used instead of auto generated codes. If there are no more usable codes, an auto generated code will be used in that case.

The csv input format:
  • [Column A] Coupon Code - can have up to 32 alphanumeric characters and must be unique per coupon
  • [Column B] Status - accepted values: Active, Inactive
  • [Column C] Admin Note - a note about the coupon or leave blank

To import gift certificate product codes, select "Add New" from the codes list.
Fields:
3.3.2.1 Product
Options
Single selection: shows all your products that have been tagged as gift certificate products.
3.3.2.2 Coupon Code
CSV Input - Text
Enter a unique coupon code of up to 32 characters
3.3.2.3 Status
CSV Input - Option
Active: the code is not used and can be assigned
Inactive: the code is not used and cannot be assigned
Value Used: the code is used (cannot assign this status in the csv file)
3.3.2.4 Admin Note
CSV Input - Text
Internal note about the coupon
3.4 Profile (fields)
A gift certificate profile is the configuration for generating an automated gift certificate after it has been purchased on the front end. You can create as many profiles as you want.

To add a new gift certificate profile, select "Add New" from the Profile list.
Fields:
3.4.1 Title
Title: the title of the profile.
3.4.2 From Name
Text: this is the from name in the email which is being sent to a customer after ordering a gift certificate. If left blank, the Store Name is used.
3.4.3 From Email
Email: this is the from email address in the email which is being sent to a customer after ordering a gift certificate. If left blank, the Store contact email is used.
3.4.4 Email Subject
Text: this is the subject in the email which is being sent to a customer after ordering a gift certificate.
3.4.5 BCC Admin
Checkbox: If checked, a blind carbon copy of the email will be sent to the email address entered in the From Email.
3.4.8 Message Type
Options
Text: automated gift certificate email should be sent as text mail
HTML: automated gift certificate email should be sent as HTML mail
3.4.9 Image Configuration
Only available in Message Type of "HTML"
Gift Certificate Image [Options]: the image used to generate the gift certificate image. The images are located at /modules/awocoupon/giftcert/images. Custom images of types .png and .jpg can be added to this folder for more selection.

The following fields are used to generate the gift certificate image
  • Coupon Code - the auto generated code
  • Value - the value of the gift certificate code
  • Expiration - the expiration date of the gift certificate
  • Free Text [1,2,3] - 3 fields with any custom text. All tags except for {image_embed} and {vouchers} can be used in these fields. Tags are listed in section 3.4.10.
The following items are what can be controlled when manipulating text within the image
  • Text: for the free text fields it is text, for expiration it is an option allowing you to select the format in which the date is displayed or not displayed at all.
  • Align [Options]: this is the alignment of the text, it can be left, middle, or right aligned.
  • Padding [Number]: this is the padding from either the left or right, depending on what is picked for Align.
  • Y-Axis [Number]: this is the height at which the text is placed.
  • Font [Options]: this is the font selection for the text. The fonts are located in /modules/awocoupon/giftcert/fonts. More fonts can be added to that folder for more selection.
  • Font Size [Number]: this is the size of the font selected
  • Font Color [Options]: this is the color of the font selected
3.4.10 Email Body
Text: this is the message in the email which is being sent to a customer after ordering a gift certificate. {vouchers} is required in message type of "Text" since it will be replaced with the gift certificate(s) ordered.
  • {store_name}
  • {siteurl}
  • {vouchers} - the gift certificate codes generated for the customer
  • {image_embed} - for html images, embed the gift certificate images into the email
  • {purchaser_first_name} - the customer's first name
  • {purchaser_last_name} - the customer's first name
  • {recipient_name} - the receiver's name
  • {recipient_email} - the receiver's email
  • {recipient_message} - the message that the customer writes when purchasing the gift certificate
  • {today_date} - the current date
  • {order_id} - the order id
  • {order_date} - the date the order was created
  • {order_total} - the order total
  • {product_name} - the name of the product being purchased. If multiple and used in email, names are comma separated
3.5 History of Uses
The history of uses displays transactions of the customer within the store.
3.5.1 Coupons
This screen displays all the customers that have used a coupon/gift certificate during the ordering process.
3.5.2 Gift Certificates
This screen displays all the gift certificates in your store including the value, value used, balance, and customer who ordered it (if any).
3.5.3 Orders
This screen displays all the orders in which gift certificates where purchased. It displays the order number and the gift certificate codes of the order.
3.6 Import
The import feature allows the user to import multiple coupons by uploading a csv file.

A sample of the file is displayed at the bottom of the screen. Each row represents one coupon. Capitalization of letters is irrelevant.
Field Values:
  • ID - must be unique per coupon
  • Coupon Code - can have up to 32 alphanumeric characters and must be unique per coupon
  • Published - published or unpublished (see section 3.2.1.3 for more details)
  • Function Type - coupon or gift certificate, shipping, buy x get y, or parent (see section 3.2.1.1 for more details)
  • Percent or Amount - percent or amount (see section 3.2.1.4 for more details)
  • Discount Type - overall or specific (see section 3.2.1.5 for more details)
  • Value - a number or leave blank if using Value definition field (see section 3.2.1.6 for more details)
  • Value Definition - see section 3.2.1.6 on how to define progressive values
  • Number of uses (Total) - total numer of time the coupon can be used (see section 3.2.1.8 for more details)
  • Number of uses (Per customer) - number of times each customer can use the coupon (see section 3.2.1.9 for more details)
  • Minimum Value
    • first column needs to be set to either overall, specific, or left blank
    • second column requires a number (if the first column has been populated)
    • see section 3.2.1.10 for more details
  • Start Date - date in YYYYMMDD format, or left blank if no start date (see section 3.2.1.11 for more details)
  • Expiration - date in YYYYMMDD format, or left blank if no expiration date (see section 3.2.1.12 for more details)
  • Customer
    • [first column] Customer Type - customer or shopper group (see section 3.2.1.17 for more details)
    • [second column]
      • for customer enter the user IDs seperated by a comma
      • for shopper group enter the shopper group IDs seperated by comma
  • Asset 1
    • first column (type): product, category, vendor, manufacturer, shipping
    • second column (mode): include or exclude the asset
    • third column (number): the number of assets (only used by Buy X get Y)
    • fourth column: comma separated IDs of the asset
  • Asset 2
    • first column (type): product, category, vendor, manufacturer, shipping
    • second column (mode): include or exclude the asset
    • third column (number): the number of assets (only used by Buy X get Y)
    • fourth column: comma separated IDs of the asset
  • Exclude Products on Special - valid values include 'Yes', 'No' or leave blank. If left blank, no is assumed (see section 3.2.1.15 for more details)
  • Exclude Gift Certificate Products- valid values include 'Yes', 'No' or leave blank. If left blank, no is assumed (see section 3.2.1.16 for more details)
  • Description - a note about the coupon or leave blank
  • Process Type - used by Parent and Buy X Get Y coupons
    • for parent: first, lowest, highest, all, allonly
    • for buy x get y: first, lowest, highest
  • Maximum Discount Qty - only used by Buy X get Y to set the maximum number of discounts
Before uploading the file, there are 2 options to consider.
- Exclude the First Row: select the checkbox if you have a header row
- Even with errors in the batch, Save coupons with no errors: This options is a mare choice of preference as some users prefer to fix errors and re-upload the whole file while others prefer to re-upload the corrected rows only. Checking the box will save any coupons that do not have errors and unchecking the box will save all coupons only if there are no errors.

To upload the csv file click the browse button, open your csv file and click the "Upload" button. Any errors will be clearly displayed on the screen, and you will be notified of any savings.
You can view the successfully imported coupons in the Coupons list.
3.7 Reports
There are six standard reports offered within AwoCoupon Pro.
3.7.1 How to run a report
1) Choose "Reports" from the menu.
2) From the Reports drop down, choose a report you want to run.
3) If you want to see all the data available without further specifiction, click the "Run Report" button. A new window will open displaying the report.
4) If you want to further narrow down the data you are interested in, make your selections under Order Details and/or Coupon Details section.
5) Once you have all the parameters selected click the "Run Report" button. A new window will open displaying the report.
3.7.2 How to export a report
1) Run a report
2) When the new window opens with your report, click on the MS Excel icon located next to the report name on the left. A new window will open offering you the option to save the report.
3) Click the save button and save the report on your computer.
Note: reports will be exported as a csv. file
3.7.3 Coupon List
This report shows you the list of coupons you have created within AwoCoupon Pro. Since it is exportable as a csv file, it can be imported back into AwoCoupon Pro, if for instance, any of your coupons were mistakenly deleted.
3.7.4 Coupon Usage vs. Total Sales
This report lets you analyze the coupon usage compared to total sales, allowing you to see how many sales were perhaps generated by your coupon promotion and the revenue these sales brought to your business. Report is supported by graphical view too.
3.7.5 Coupon Usage vs. Location
This report lets you evaluate customers relative to their geographical location and their responsiveness to a specific coupon promotion. This proves to be helpful if your marketing campaigns are location specific, as it helps you to target a local consumer base more effectively.
3.7.6 History of Uses - Coupons
This report displays all the customers that have used a coupon during the ordering process. It is easy to analyze user's tendency to take advantage of coupon promotions and determine customer loyalty.
3.7.7 History of Uses - Gift Certificates
This report displays all the gift certificates within your store. With this report, you can view the value and balance left on all gift certificates.
3.7.8 Purchased Gift Certificate List
This report displays all the gift certificates that were purcased from your storefront. With this report, you can determine the user details of the purchaser.
3.8 Configuration
3.8.1 Enable Store Coupons
This option allows you to use Prestashop Coupons along with AwoCoupon. When a code is put in, it is first run through AwoCoupon to see if it exists. If the code is not found, then the Prestashop Coupon process is called. AwoCoupon takes precedence over the store Coupon. If the code exists within AwoCoupon (no matter if it is published or not), then the code is processed through the AwoCoupon system. Also note that codes processed through the Prestashop Coupon System will be limited to the flexibility of Prestashop Coupons.
3.8.2 Calculate the discount before tax (Gift Certificates)
This option allows you to enable true discount before tax for gift certificates. Set to "yes" for discount before tax and "no" for discount after tax.
3.8.3 Calculate the discount before tax (Coupons)
This option allows you to enable true discount before tax for coupons. Set to "yes" for discount before tax and "no" for discount after tax.
3.8.4 Enable Multiple Coupons
This option allows you to enable the use of multiple coupons. Entering an integer in the text field will limit the number of coupons that can be accepted. If for example 2 is entered, then the maximum number of coupons that will be accepted at once will be 2.
3.8.5 Delete expired coupons x days after expiration
This option enables expired coupons to be automatically deleted. You can set the offset in days. So if you set the offset for 3 days, then all coupons that expire today will be deleted in 3 days.
3.8.6 Case Sensitive Coupon Code
This option allows you to change the case sensitivity of coupon codes. By default coupon codes are case sensitive.
3.8.7 CSV Delimiter
This option allows you to change the delimiter of csv files. This would work for both importing and exporting (through reports). By default a comma is used as the delimiter. You have the option to change it to a semi-colon.
3.8.8 Gift Certificate Products
Configuration for gift certificate products.
3.8.8.1 Vendor Activate
If this feature is activated then an email is sent to the Vendor notifying them of the voucher purchase at the same time the voucher is emailed to the customer. For this to work, the vendor email has to be setup within the gift certificate product.
3.8.8.2 Vendor Email Subject
The subject of the vendor email. You can use dynamic tags in Email Body.
3.8.8.3 Vendor Email Body
The message being sent to the vendor. The dynamic tags include:
- {vendor_name}
- {vouchers}
- {purchaser_first_name}
- {purchaser_last_name}
- {today_date}
- {order_id}
3.8.8.4 Vendor {vouchers}
This field allows you to format the {vouchers} tag. There can be multiple vouchers and this lets you format it in the way you like. By default the format of the vouchers tag is:
[code1] - [price1] - [product name]
[code2] - [price2] - [product name]
....
You can also use any of the dynamic tags in Email Body.
3.8.9 Custom Error Messages
With the default installation of AwoCoupon Pro, the error message seen by the customer if a coupon is rejected is the default message: "Coupon code not found" You might want to change this note to display a more specific message. For example, there are coupons which require the user to login before it can be used. Instead of displaying the default message, you can change it to a more appropriate text such as "Please Login to use this coupon". To do this, click in the error display box, type your message and press the save button.
4 Example Usage
The customer and product field are multiple selection boxes. So you can select none, one, or multiple customers and/or products for every coupon configured.
4.1 Example 1
Function Type: Coupon
Coupon Code: COUPONCODE1
Published: Published
Percent or Amount: Percent
Discount Type: Overall
Value: 25
Value Definition: blank
Number of Uses Total: blank
Number of Uses Per Customer: 1
Minimum Value: blank
Expiration: blank
Customer: None chosen
Asset: None chosen
This coupon will work for all products and all customers (including any new registrations) for exactly 1 time per customer. It will take 25% off the total value of the shopping cart. The coupon never expires. The customer is required to have an account to be able to use this code.
4.2 Example 2
Function Type: Coupon
Coupon Code: COUPONCODE2
Published: Published
Percent or Amount: Amount
Discount Type: Overall
Value: 25
Value Definition: blank
Number of Uses Total: blank
Number of Uses Per Customer: blank
Minimum Value: 100
Expiration: 2020-10-17
Customer: 2 chosen
Asset Type: Category
Asset Selection: Include 3 chosen
This coupon will work for the 2 customers chosen as long as their shopping cart total has a minimum total of 100 and at least one product in the cart is in the category of one of the 3 chosen categories in the coupon. It will take $25 off the total value and it is a permanent coupon so it will work everytime used as long as the criteria are met. The coupon will become invalid on October 17, 2020.
4.3 Example 3
Function Type: Coupon
Coupon Code: COUPONCODE3
Published: Published
Percent or Amount: Percent
Discount Type: Specific
Value: blank
Value Definition: 2-10;5-20;7-25;
Number of Uses Total: blank
Number of Uses per Customer: 3
Minimum Value: 120
Expiration: blank
Customer: 7 chosen
Asset Type: Product
Asset Selection: Include 3 chosen
This coupon will work for the 7 customers chosen as long as their shopping cart has a minimum total of 120 and at least one product in the cart is one of the 3 chosen in the coupon. Since the discount type is set to specific, the percentage discount only applies to the products listed in the coupon. If a customer has 1 of any of the 3 products, he/she would get no discount. If the customer has 2 of any of the 3 products, a discount of 10% will be given for the 2 products. If the customer has 5 (total quantity) of any of the 3 products, a discount of 20% will be given for the 5 products. And lastly if the customer has 7 or more of the 3 products, a discount of 25% will be given on the 7 or more products. Each of the customers selected can use this coupon a total of 3 times. Since it has no expiration date, it never expires, but with the restrictions set, it effectively expires after all 7 customers use the coupon on 3 seperate occasions each.
4.4 Example 4
Function Type: Shipping
Coupon Code: COUPONCODE4
Published: Published
Percent or Amount: Amount
Value: 5
Number of Uses Total: 100
Number of Uses Per Customer: blank
Minimum Value: blank
Expiration: blank
Customer: blank
Asset Selection: Exclude 2 chosen
Anybody can use this coupon (an account is not required) and it will work for exactly 100 times. This is great for campaigns saying the first 100 orders get "fill in the blank". The total value in products bought has to be 100 or over. As long as the shipping selected is one of the ones supported and it is NOT one of the ones selected in the coupon, $5 will be taken off of the shipping cost. If the cost of shipping is less than $5 then the cost of shipping will be used instead.
4.5 Example 5
Function Type: Gift Certificate
Coupon Code: COUPONCODE5
Published: Published
Value: 35
Expiration: blank
This creates a gift certificate coupon. Gift certificate coupons only deduct from the products, not the shipping cost. This coupon can be used as many times as is needed to use up the value of 35.
5 Troubleshooting
If you are having problems with coupons being accepted, there are a few things you can try:
  • Go to AwoCoupon->Configuration->Error tab and change the error messages to something more identifiable, to see why a coupon fails.
  • FTP into your webroot and verify these files are within your override folder:
    • /override/classes/Cart.php
    • /override/classes/Order.php
    • /override/classes/Discount.php
    • /override/controllers/ParentOrderController.php
    These files would have been added when you first installed AwoCoupon. If you already had these override files, your originals were saved with the postfix .awocoupon.php. You will need to merge the files in order to get your original function back.
